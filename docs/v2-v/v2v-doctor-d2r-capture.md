# V2-V Doctor D2R Capture

Doctor is valuable because it exercises a different fat-JAR/CDS protocol. The
diagnostic capture must use the exact confirmed D2R artifact and a fresh D2R
CDS archive. The identity record therefore includes artifact SHA, CDS archive
SHA, image ID, CDS mapping proof, launch mode, runtime policy, and reducer
engine.

The capture is separate from the existing V2-K memory pairs. Reusing those
pairs or reusing a CDS archive trained from another artifact would invalidate
the generated-family join.

Current repository state: Doctor has no fresh complete matched lifecycle bundle.
