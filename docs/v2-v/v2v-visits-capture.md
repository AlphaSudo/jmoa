# V2-V Visits Capture

Visits has strong public V2-L runtime evidence, but that evidence is not a
generated-family lifecycle bundle. The V2-V campaign must first create a
static generated-class inventory for one explicitly selected visits artifact,
then capture startup, warmup, and workload diagnostics for that same artifact.

The selected universe must be recorded as either baseline or baseline plus raw
reducer. Existing V2-L memory evidence is not reused as generated-family
evidence.

Current repository state: the fresh matched visits bundle is incomplete.
