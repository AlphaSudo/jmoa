# V2-P Golden Workflow

The required order is:

1. Reducer recommendation.
2. Runtime-policy recommendation.
3. SHA-backed preflight.
4. Artifact and materialization proof.
5. Baseline and candidate semantic smoke.
6. One paired screen.
7. Three-pair confirmation.
8. V2-C validation.
9. V2-D attribution.
10. Reviewed claim-register update.

No later gate is allowed to repair or bypass an earlier failure. A screen is
not confirmation, and a workflow replay is not fresh runtime evidence.
