# V2-T Family ROI Ranking

V2-T keeps ROI ranking subordinate to evidence and safety. Static bytes and
class counts can prioritize the next diagnostic capture, but cannot admit a
prototype on their own.

1. Lambda/metafactory: runtime-only implementations were observed in the
   incomplete customers workload capture; existing JMOA lambda work does not
   make this a new generated-class mutation target.
2. Spring Data generated helpers: meaningful static surface in customers and
   Doctor, but no fingerprint-matched lifecycle relevance or semantic proof.
3. Spring AOT BeanDefinitions: static Doctor surface only; report-only pending
   matched capture and bean-registration semantic proof.
4. CGLIB, JDK proxy, ByteBuddy, and Hibernate proxy: remain mutation-blocked
   even if runtime-relevant because proxy/identity/interception semantics are
   not discharged.

No item is an admitted prototype candidate in this phase.
