# V2 Historical Phase Index

The public documentation is organized by capability. This index preserves the
original phase chronology for provenance and detailed audits.

| Phases | Capability record |
| --- | --- |
| V2-A | Generated/synthetic inventory, attribution, and safety taxonomy |
| V2-B | Classfile footprint, near-64KB risk, and runtime correlation |
| V2-C | Evidence validation, pairing, variance, and historical replay |
| V2-D | smaps/NMT/heap/class memory attribution |
| V2-E to V2-L | Reducer implementation, hardening, raw-engine recovery, and service confirmations |
| V2-M to V2-P | Reducer recommendation, runtime-policy recommendation, preflight, and workflow coordination |
| V2-Q | Application-class reducer admission and rejected runtime confirmation |
| V2-R to V2-W | Generated-family ROI, relevance, matched lifecycle evidence, and no-admission decision |

The detailed records remain in `docs/v2-a` through `docs/v2-w`. Phase closure
labels describe development maturity; they are not additional performance
claims. Current claims come only from the [Claim Register](../v2-claim-register.md)
and [Final Matrix](../results/v2-three-service-matrix.md).
