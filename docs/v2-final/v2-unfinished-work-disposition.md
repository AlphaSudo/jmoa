# V2 Unfinished-Work Disposition

Nothing in the V2 plan remains ambiguously `PARTIAL`, `OPEN`, or `BLOCKED`.
Every capability has one terminal release disposition.

## SHIPPED

- V1 full-P2 optimizer and deployment materialization.
- Raw dependency LVT/LVTT reducer, byte-preservation auditor, and manifest v2.
- Classfile/near-64KB profiler, V2-C evidence, and V2-D attribution engines.
- Reducer/runtime-policy recommendations, preflight, workflow automation,
  materialization proof, semantic smoke, screen, and confirmation workflow.
- Documented no-CDS and CDS runtime-policy support.

## VALIDATED_BUT_NOT_SHIPPED

- Generated-class inventory, runtime attribution, safety, ROI, lifecycle capture,
  and matched-evidence tooling remain analysis-only.
- Application-class reduction passed artifact safety but not runtime promotion.
- Doctor confirmation is useful private evidence, not a public product workflow.

## INVESTIGATED_AND_REJECTED

- V2-H hardened ASM promotion failed its runtime screen and was replaced by the
  byte-preserving raw engine.
- V2-Q application-class promotion failed confirmation and had only 480 bytes
  of artifact ROI.
- Generated-family mutation admitted no bounded safe candidate for V2.

## DEFERRED_TO_V3

- Proxy/Spring AOT/Spring Data mutation; large-method splitting; constant-pool
  and BootstrapMethods rewriting; broad AppCDS/Leyden and allocation-profiler
  matrices; automatic runtime/deployment mutation; FAST_STARTUP; Kubernetes;
  additional benchmark services; and a dashboard.

Deferred items are not V2 release promises. The V2 repository retains the
analysis needed to revisit them without enabling unsafe mutation.
