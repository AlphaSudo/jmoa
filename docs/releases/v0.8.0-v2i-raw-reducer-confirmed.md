# v0.8.0-v2i-raw-reducer-confirmed

JMOA V2-I adds an explicit raw bytecode reducer engine for the existing
LocalVariableTable / LocalVariableTypeTable reducer.

## What Changed

```text
jmoa.reducer.engine=raw
```

The raw engine preserves BootstrapMethods while removing only nested
local-variable debug tables from method `Code` attributes. It keeps the V2-F jar
safety policy:

```text
signed jars skipped
multi-release jars skipped
sealed jars skipped
module-info.class preserved
```

The default engine remains:

```text
jmoa.reducer.engine=asm
```

## PetClinic Confirmation

V2-I was confirmed on Spring PetClinic customers-service under the documented
public no-CDS exploded Boot protocol:

```text
comparison: full P2 vs full P2 + V2-I raw reducer
launch mode: EXPLODED_BOOT_APP / JarLauncher
runtime policy: NO_CDS_LOW_DIRTY
MALLOC_ARENA_MAX: 1
CDS/AppCDS/Leyden: off
runtime javaagent: absent
valid runs: 6/6
paired wins: 2/3
median PSS delta: -4,467 KB
median Private_Dirty delta: -4,208 KB
median memory.current delta: -4,493,312 bytes
```

V2-D attribution identified heap page-touch reduction and anonymous_rw allocator
reduction as the main explanations. The result is not claimed as retained-object
shrinkage or class-count savings alone.
