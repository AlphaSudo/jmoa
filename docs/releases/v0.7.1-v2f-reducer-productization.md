# v0.7.1-v2f-reducer-productization

This release closes V2-F as reducer productization for the V2-E LVT/LVTT reducer.

## What Changed

V2-F makes the reducer safer and more auditable:

```text
signed JARs are skipped by default
multi-release JARs are skipped by default
sealed JARs are skipped by default
module-info.class is preserved
classes carrying BootstrapMethods remain skipped in mutation mode
reducer manifest includes input/output SHA-256 hashes
ZIP timestamp policy is recorded as preserve
```

Preserved attributes:

```text
LineNumberTable
SourceFile
StackMapTable
RuntimeVisibleAnnotations
RuntimeInvisibleAnnotations
Signature
BootstrapMethods
module-info.class
```

## Claim Boundary

V2-F does not add a runtime memory claim. The only confirmed V2-E runtime claim
remains PetClinic customers-service under EXPLODED_BOOT_APP no-CDS policy.

Not claimed:

```text
Doctor runtime win
fat-JAR runtime win
startup win
cross-service runtime generalization
new reducer type
broader metadata stripping
```
