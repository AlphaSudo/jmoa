# V2-S Family Memory Correlation

V2-S joins existing V2-D reports only as context. It does not treat a static family footprint or a histogram association as causal memory proof.

The available PetClinic customers capture makes lambda runtime generation observable, but the previous raw-reducer confirmations remain attributed to native/metaspace/anonymous mapping behavior rather than a generated-family mutation. Spring Data remains `NO_OBSERVED_RUNTIME_CORRELATION` in the available customers capture. Doctor and visits are `UNEXPLAINED` for family-specific correlation because the necessary matched captures are incomplete.
