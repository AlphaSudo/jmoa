# V2-S Static-To-Runtime Reconciliation

The report carries each number's universe: static packaged class entries, diagnostic class-load events, or post-workload histogram objects. They are not interchangeable.

PetClinic customers has `RUNTIME_GENERATED` lambda evidence. Its Spring Data, ByteBuddy, Hibernate, and bridge surfaces are `STATIC_ONLY` in the available diagnostic capture. Doctor families are `RUNTIME_RELEVANCE_UNKNOWN` because a paired diagnostic class-load capture is unavailable.

This phase does not infer that a static class is unused in every workload; it records the boundary of the available capture.
