The ranking is an investigation queue, not a mutation queue.

1. Lambda call-site/runtime class shape: large static surface and runtime generation observed, but it is already within JMOA's original lambda domain and no new V2-S mutation is admitted.
2. Spring Data generated helpers: about 1.38 MB static context in both customers and Doctor inventories, but no relevance proof sufficient for a family prototype.
3. Synthetic/bridge-heavy classes: large static footprint but no safe consolidation concept or family-specific semantic model.
4. Proxy/AOT families: blocked even where runtime relevance could later be established.
