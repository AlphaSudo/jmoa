# V2-O Runtime Policy Automation

V2-O turns the V2-N report-only policy decision into explicit workflow helpers.
It prepares preflight, CDS training records, materialization proof, semantic
smoke, paired capture, V2-C validation, and V2-D attribution inputs.

V2-O is automation, not optimization. It does not enable the reducer, select a
runtime policy automatically, train a CDS archive without an explicit command,
or create a runtime claim.

Runtime claims remain subject to V2-C validation and V2-D attribution.
